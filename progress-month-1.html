Project Avocado: Building AI in the Garage - Month 1 Recap

From Bare Metal to AI Agent Platform (Jan 24 - Apr 6, 2025)

What happens when you decide to build a cutting-edge AI platform not in a well-funded corporate lab, but on your own hardware, in your own space? That's the core idea behind Project Avocado, an ambitious venture launched by Terry Dynamics just over a month ago. The goal: develop a powerful, self-hosted AI system capable of complex reasoning and interaction, exploring the full stack from bare metal up.

This project goes beyond simply running pre-trained models. It's about understanding the entire process: installing Arch Linux, configuring networks, wrangling GPU drivers, building custom APIs, and diving into the developing field of AI agents.

Key Milestones: The First Month

The initial weeks involved transforming a blank server into a functional AI hub:

  • Foundation & First Models: Starting with Arch Linux, we established core infrastructure (networking, SSH, NVIDIA drivers, CUDA, Docker). This allowed us to run early models like DeepSeek-Coder 6.7B locally using llama-cpp-python on an initial GTX 1070 Ti.
  • Platform Expansion: We integrated Mistral-7B via transformers and adopted Ollama for easier model serving, adding models like Google's Gemma 3 12B.
  • Interfaces & Access: A Discord bot ("Avocado") was created for LLM interaction. The Terry Dynamics AI Lab web dashboard began taking shape, requiring Nginx configuration, CORS troubleshooting, and integrating UIs for Ollama and ComfyUI (image generation) under api.terrydynamics.com.
  • Hardware & Agent Exploration: A significant upgrade to an NVIDIA P40 GPU (24GB VRAM) enabled larger models. We also took initial steps with the OpenManus agent framework, connecting it to our local Ollama instance for basic tool use.
  • Refinement & Roadblocks: Recent work focused on improving the API service (concurrency, sessions), establishing a GitHub workflow, and tackling challenges like a paused RAG implementation attempt, DNS updates due to IP changes, and ongoing efforts to utilize a second GPU (RTX 3060 Ti).

Tech Stack Highlights

  • OS: Arch Linux (Bare Metal)
  • Core GPU: NVIDIA P40 (24GB VRAM)
  • Model Serving: Ollama, llama-cpp-python
  • Models: Gemma 3 12B, Mistral 7B, DeepSeek-Coder 6.7B (and others)
  • Backend API: Python (Flask), managing sessions and concurrency
  • Web Server: Nginx (Reverse Proxy)
  • Containerization: Docker (with GPU support)
  • Agent Framework: OpenManus (Initial integration)
  • Interfaces: Custom Discord Bot, Web Dashboard (HTML/JS), Ollama WebUI, ComfyUI

Challenges of Building from Scratch

This hands-on approach inevitably presents unique hurdles:

  • Environment Management: Aligning Python virtual environments with system packages.
  • Hardware Constraints: Working within VRAM limits (leading to the P40 upgrade) and managing data center card thermals.
  • Configuration Nuances: Debugging Nginx proxies, CORS, systemd services, and Docker networking.
  • Tooling Integration: Adapting frameworks like OpenManus often requires custom code.
  • Infrastructure Stability: Handling external factors like dynamic IP addresses and DNS propagation.
  • Multi-GPU Complexity: Initializing and utilizing multiple GPUs with different architectures.

Current Status & The Road Ahead

After an intensive start, Project Avocado now features a stable server with multiple LLMs accessible via a custom API, a Discord bot, and various web UIs. The foundational work is largely complete.

The next phase shifts focus towards the core vision: **agent development**. Key goals include:

  • Utilizing Gemma 3 and other capable models within the agent framework.
  • Defining more complex tasks and tools for the agent.
  • Improving system architecture for better model management and resource allocation.
  • Resolving the multi-GPU setup to potentially offload specific tasks.

Project Avocado continues to be an exploration into building powerful, self-hosted AI. Stay tuned for future updates!
